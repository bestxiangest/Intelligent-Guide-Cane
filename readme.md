# 星辰引路者——基于ESP32-S3的多模态智能导盲系统

![智能导盲杖](https://img.shields.io/badge/星辰引路者-ESP32--S3-blue)
![版本](https://img.shields.io/badge/版本-1.0.0-green)
![许可证](https://img.shields.io/badge/许可证-MIT-orange)

## 项目概述

星辰引路者是一款基于ESP32-S3微控制器的多模态智能导盲系统，旨在帮助视障人士更安全、便捷地出行。该系统集成了多种传感器和人工智能技术，能够实时检测周围环境，识别障碍物、盲道和红绿灯，并通过语音交互提供导航和天气查询等功能。

### 补充说明

微信小程序、硬件端、服务端三个压缩包为全部项目工程文件，解压缩后使用。其他内容为硬件端esp32的程序源码.

## 功能特点

- **障碍物检测**：使用超声波传感器实时检测前方障碍物，并根据距离提供不同级别的警报
- **盲道识别**：通过摄像头实时识别地面盲道，辅助视障人士沿着正确的路径行走
- **红绿灯识别**：自动识别交通信号灯状态，提醒用户何时安全通行
- **语音交互**：支持语音指令输入和语音反馈，实现无障碍人机交互
- **智能导航**：通过语音指令设置目的地，提供实时导航指引
- **天气查询**：支持通过语音查询指定城市的天气情况
- **大模型对话**：集成大语言模型，支持自然语言对话和信息查询

## 硬件要求

- ESP32-S3开发板（推荐ESP32-S3-DevKitC-1）

- 蜂鸣器
- 按钮（用于触发语音交互）
- 电源模块（锂电池+充电管理）

## 使用方法

1. 开机后，系统将自动连接WiFi并初始化各个模块
2. 超声波传感器会持续检测前方障碍物，当检测到障碍物时会通过振动和蜂鸣器提醒用户
3. 摄像头会实时检测盲道和红绿灯，并在必要时提供警报
4. 按下语音按钮可以进行语音交互：

- 导航指令：例如"导航到华东交通大学"
- 天气查询：例如"今天南昌天气怎么样"
- 一般问题：可以询问任何问题，系统会通过大模型提供回答

## 系统架构

系统基于FreeRTOS多任务架构，主要包含以下任务：

- 超声波检测任务：负责障碍物检测
- 视觉检测任务：负责盲道识别
- 红绿灯检测任务：负责交通信号灯识别
- 语音处理任务：负责语音交互
- 警报任务：负责整合各类警报并通过振动和声音提醒用户
- 导航任务：负责提供路径导航

## API集成

本项目集成了多个外部API服务：

- 百度语音识别API：用于将语音转换为文本
- 和风天气API：用于获取天气信息
- AnythingLLM API：用于大模型对话
- 高德地图API：用于地理编码和路径规划

## 三、各模块接线说明

### 1. 超声波测距（HC-SR04）

用于前方障碍物检测，触发振动与蜂鸣器报警。

| HC-SR04 引脚 | 接 ESP32-S3 | 说明 |
|-------------|-------------|------|
| VCC         | 5V 或 3.3V | 建议 5V 提高测距稳定性 |
| GND         | GND        | 共地 |
| Trig        | **GPIO 8** | 触发测距 |
| Echo        | **GPIO 18**| 回响脉冲输出 |

- 有效测距范围：约 2 cm～400 cm  
- 代码中障碍物报警阈值：20 cm（可在 `config.h` 中修改 `OBSTACLE_DISTANCE_THRESHOLD`）

---

### 2. 振动模块 + 蜂鸣器

| 模块       | 模块引脚 | 接 ESP32-S3 | 说明 |
|------------|----------|-------------|------|
| 振动马达   | 正极/VCC | **GPIO 3**  | 高电平振动（建议经三极管或模块驱动） |
| 振动马达   | GND      | GND         | 共地 |
| 蜂鸣器     | 正极/I/O | **GPIO 11** | 代码中为**低电平**响，高电平关闭 |
| 蜂鸣器     | GND      | GND         | 共地 |

- 蜂鸣器若为有源蜂鸣器，建议串联限流电阻或使用驱动电路。  
- 振动模块若电流较大，请勿直连 GPIO，用三极管或现成马达驱动模块。

---

### 3. 测试/语音按钮

| 按钮引脚 | 接 ESP32-S3 | 说明 |
|----------|-------------|------|
| 一脚     | **GPIO 2**  | 代码使用 `INPUT_PULLUP`，按下接 GND |
| 另一脚   | GND         | 按下时与 GPIO 2 导通 |

- 按下为低电平，松开为高电平（内部上拉）。

---

### 4. 光敏传感器 + 照明灯

代码中为**数字型**光敏（DO 输出），暗时 DO 为高电平并自动打开照明。

| 光敏模块引脚 | 接 ESP32-S3 | 说明 |
|-------------|-------------|------|
| VCC         | 3.3V       | 供电 |
| GND         | GND        | 共地 |
| DO（数字输出）| **GPIO 10**| 暗=HIGH，亮=LOW |
| AO          | 不接       | 代码未使用模拟量 |

| 照明灯/继电器 | 接 ESP32-S3 | 说明 |
|-------------|-------------|------|
| 控制端/IN   | **GPIO 21**| 高电平开灯 |
| GND         | GND        | 共地 |

- 若照明为大电流负载，请通过继电器或 MOS 管驱动，勿直连 GPIO。

---

### 5. 麦克风 INMP441（I2S 输入）

用于唤醒词检测与语音识别，连接 I2S 从设备（左声道）。

| INMP441 引脚 | 接 ESP32-S3 | 说明 |
|-------------|-------------|------|
| SCK         | **GPIO 4**  | I2S 位时钟 (BCK) |
| WS          | **GPIO 5**  | I2S 字选择 (LRC) |
| SD          | **GPIO 6**  | 数据输出 → ESP32 数据输入 |
| L/R         | GND         | 接 GND 选左声道（代码用左声道） |
| VDD         | 3.3V        | 供电 |
| GND         | GND         | 共地 |

- 代码中 I2S 输入使用 `I2S_NUM_0`，16kHz、16bit、单声道。

---

### 6. 扬声器 MAX98357A（I2S 输出）

用于语音播报与提示音。

| MAX98357A 引脚 | 接 ESP32-S3 | 说明 |
|----------------|-------------|------|
| BCLK           | **GPIO 15** | I2S 位时钟 |
| LRC            | **GPIO 16** | I2S 字选择（左右声道） |
| DIN            | **GPIO 7**  | 数据输入 ← ESP32 数据输出 |
| VIN            | 5V 或 3.3V  | 供电（5V 音量更大） |
| GND            | GND         | 共地 |
| SD（关机）     | 不接或接高  | 接高为工作，接低关机 |
| GAIN           | 按模块说明  | 可选接 GND/悬空等调节增益 |

- 代码中 I2S 输出使用 `I2S_NUM_1`，与麦克风分属不同 I2S 端口。

---

### 7. GPS 模块（需自行改串口）

当前代码中 **GPS 与串口调试共用 `Serial`（USB 串口）**，因此：

- 若只跑程序、不接外部 GPS：可忽略 GPS 接线，仅 WiFi/语音/超声波等可用。  
- 若接**外部 GPS 模块**（如 NEO-6M、NEO-8N 等）：  
  - 必须改用**硬件串口**（如 `Serial1`），不能继续用 `Serial`。  
  - GPS 的 **TX** 接 ESP32-S3 的 **RX**（例如 GPIO 17 若作 Serial1 RX），  
  - GPS 的 **RX** 可接 ESP32 **TX** 或悬空（仅接收 NMEA 时可不接）。  
  - 在 `gps.h` 中把 `GpsSerial` 改为 `Serial1`，并在 `gpsInit()` 里用 `Serial1.begin(9600)` 或与模块一致的波特率（当前代码写 115200，请按模块规格设置）。

**示例（仅作参考，具体引脚以你使用的 Serial1 为准）**：

| GPS 模块引脚 | 接 ESP32-S3 | 说明 |
|-------------|-------------|------|
| VCC         | 3.3V        | 供电 |
| GND         | GND         | 共地 |
| TX（GPS 发送）| **Serial1 的 RX**（如 GPIO 17）| 接收 NMEA 数据 |
| RX（GPS 接收）| 可选接 Serial1 的 TX | 若不需要向 GPS 发指令可悬空 |

修改代码后需在 `config.h` / `gps.h` 中确认串口与波特率与模块一致。

## 六、API / 服务端地址说明（与 web 端配合）

硬件端通过 HTTP 访问**自建服务端**的地址如下（仅 AI 对话、GPS、导航；**语音合成不经过自建服务器**）。如需部署自己的服务端，请在下列文件中替换 IP 与端口：

| 用途     | 当前代码中的地址 | 所在文件（约） |
|----------|------------------|----------------|
| AI 对话  | `http://my_ip:12345/ai` | network.cpp |
| GPS 上报 | `http://my_ip:12345/gps` | network.cpp |
| 导航更新 | `http://my_ip:12345/navigation_update` | main.cpp |
| 退出导航 | `http://my_ip:12345/exit_navigation` | main.cpp |

- **语音合成**：当前实现为**硬件端直接请求百度云语音合成 API**（`voice.cpp` 中的 `baiduTTS_Send`，请求 `https://tsn.baidu.com/text2audio`），**不经过自建服务器**。只需在 `config.h` 中配置 `BAIDU_CLIENT_ID`、`BAIDU_CLIENT_SECRET`，并在运行时通过百度接口获取 access_token 即可。服务端（`web/app.py`）未提供、也未使用语音合成接口。
- 克隆项目后若使用**自己的服务器**：在 `src/network.cpp` 和 `src/main.cpp` 中搜索 `my_ip`，将 IP 与端口改为你的服务地址（端口 12345）。  
- **百度语音**（识别 + 合成）均在硬件端通过百度云 API 完成，在 `config.h` 中配置自己的 `BAIDU_CLIENT_ID`、`BAIDU_CLIENT_SECRET` 等即可。

## 七、用手机热点 + 电脑当服务器（局域网使用，无公网服务器）

没有云服务器时，可以用**手机开热点**，让**电脑**和 **ESP32** 都连到这个热点，电脑上运行项目里的服务端程序，ESP32 访问电脑的 IP 即可。下面按「小白」视角一步步写，照着做即可。

### 7.1 整体思路（先看明白再动手）

1. 手机开一个 **WiFi 热点**（比如名称叫 `我的热点`，设一个密码）。
2. **电脑**连上这个热点（用来跑服务端）。
3. **ESP32** 也连上同一个热点（在代码里填热点的名称和密码）。
4. 在电脑上查到**本机在当前热点下的 IP 地址**（例如 `192.168.43.123`）。
5. 把硬件端代码里**访问自建服务器的地址**（AI、GPS、导航）都改成这个 **电脑 IP**，端口 **12345**。
6. 电脑上运行**服务端**（`web` 目录下的 Python 程序），ESP32 烧录并上电，即可在局域网内对话、导航等（语音合成仍由 ESP32 直接请求百度云，不经过电脑服务端）。

---

### 7.2 第一步：手机开热点

1. 打开手机 **设置** → **移动网络**（或「网络和互联网」）→ **个人热点** / **热点**。
2. 打开「**便携式 WLAN 热点**」或「**个人热点**」。
3. 记下：
   - **网络名称（SSID）**：例如 `MyPhone_XXXX`
   - **密码**：自己设一个，后面填到 ESP32 和电脑连接时要用。

> 建议：热点名称和密码尽量用**英文+数字**，避免中文或特殊符号，减少兼容问题。

---

### 7.3 第二步：电脑连上热点

1. 在电脑上打开 **WiFi 列表**，找到刚才的热点名称，点击连接，输入密码。
2. 确认电脑已经连上该热点（系统托盘或设置里会显示已连接）。

---

### 7.4 第三步：查电脑在热点下的 IP 地址

ESP32 要访问的是「电脑在当前 WiFi（热点）下的 IP」，必须在这一步查到的才有效。

**Windows：**

1. 按 `Win + R`，输入 `cmd`，回车，打开命令提示符。
2. 输入下面命令后回车：
   ```text
   ipconfig
   ```
3. 在结果里找到 **「无线局域网适配器 WLAN」** 或 **「WLAN」** 那一块（不是「以太网」）。
4. 看其中的 **「IPv4 地址」**，一般是 `192.168.43.xxx` 或 `192.168.xxx.xxx`，例如：
   ```text
   IPv4 地址 . . . . . . . . . . . . : 192.168.43.123
   ```
5. **把这个 IP 抄下来**（例如 `192.168.43.123`），后面改代码时全部替换成这个。

**macOS：**

1. 打开 **系统设置** → **网络** → **Wi-Fi** → **详细信息**（或当前连接的热点）。
2. 在 **TCP/IP** 里看 **IP 地址**；或在终端执行 `ifconfig | grep "inet "`，看对应 WiFi 接口的 `inet` 地址。

**说明：** 每次重新连热点，电脑的 IP 可能会变，所以若以后连不上了，要重新用上面方法查一次 IP，并重新改一次代码里的地址（或见下文「可选：减少 IP 变化」）。

---

### 7.5 第四步：改硬件端代码里的「服务器地址」

只需改 **2 个文件**，把 ESP32 访问自建服务器的地址从原来的公网 IP 改成**你电脑的 IP**，端口 **12345**。  
（语音合成在硬件端直接请求百度云，不经过自建服务器，无需修改。）

**1）`硬件端/src/network.cpp`**

- 搜索 `my_ip`，会看到两处：
  - 一处是 **AI 对话**的地址（`/ai`），
  - 一处是 **GPS 上报**的地址（`/gps`），  
  把这两处的 IP 都改成你的电脑 IP，端口保持 **12345**。  
  例如电脑 IP 是 `192.168.43.123`，则改成：
  - `http://192.168.43.123:12345/ai`
  - `http://192.168.43.123:12345/gps`

**2）`硬件端/src/main.cpp`**

- 搜索 `my_ip`，会看到两处：
  - **导航更新**：`/navigation_update`
  - **退出导航**：`/exit_navigation`  
  同样把 IP 改成电脑 IP，端口 **12345**。  
  例如：`http://192.168.43.123:12345/navigation_update` 和 `http://192.168.43.123:12345/exit_navigation`。

**小结：**  
以上 2 个文件里，所有 `my_ip` 都改成你的电脑 IP，端口均为 **12345** 即可。

---

### 7.6 第五步：改 WiFi 名称和密码（让 ESP32 连上热点）

1. 打开 **`硬件端/src/config.h`**。
2. 找到下面两行：
   ```c
   #define WIFI_SSID "sharp_caterpillar"
   #define WIFI_PASSWORD "wifi_password"
   ```
3. 改成你手机热点的名称和密码（用英文双引号包起来），例如：
   ```c
   #define WIFI_SSID "MyPhone_XXXX"
   #define WIFI_PASSWORD "你设的热点密码"
   ```
4. 保存文件。

---

### 7.7 第六步：在电脑上安装并运行服务端

服务端是项目里的 **Python 程序**，在 **`web`** 目录下（若仓库里是「硬件端」单独一份，就在 `硬件端/web` 下）。

**1）安装 Python**

- 若未安装：到 [python.org](https://www.python.org/downloads/) 下载 Python 3，安装时勾选 **「Add Python to PATH」**。

**2）在「命令提示符」或「终端」里进到 web 目录**

例如项目在 `D:\Intelligent-Guide-Cane`，则：

- Windows：  
  ```text
  cd D:\Intelligent-Guide-Cane\硬件端\web
  ```
  或如果服务端在仓库根目录的 `web`：
  ```text
  cd D:\Intelligent-Guide-Cane\web
  ```

（具体路径以你 clone 下来的为准，总之要进到包含 `app.py` 的那个目录。）

**3）安装依赖**

在**同一目录**下执行：

```text
pip install flask requests dashscope
```

若提示没有 `pip`，可试：`python -m pip install flask requests dashscope`。

**4）运行服务端**

在同一目录下执行：

```text
python app.py
```

若正常，会看到类似：

```text
 * Running on all addresses (0.0.0.0)
 * Running on http://127.0.0.1:12345
 * Running on http://192.168.43.123:12345
```

说明服务端已在 **12345** 端口监听，且对局域网（热点）内其他设备可见。**不要关闭这个窗口**，保持运行。

**5）可选：检查防火墙**

若 ESP32 连上热点后仍访问不到电脑，可能是 Windows 防火墙拦截了：

- 打开 **Windows  Defender 防火墙** → **高级设置** → **入站规则** → **新建规则**；
- 选择「端口」→ 下一步 → TCP，特定本地端口填 **12345** → 允许连接 → 下一步 → 全选（域/专用/公用）→ 命名如「导盲杖服务端」→ 完成。

### 7.8 第七步：烧录 ESP32 并上电

1. 用 USB 线把 ESP32 接到电脑，用 PlatformIO 或 Arduino 打开硬件端工程，**编译并烧录**（已按上面改好 WiFi 和服务器地址）。
2. 烧录完成后，可以拔掉 USB，用**充电宝或 5V 电源**给 ESP32 供电（若你接线是外供电方案）。
3. ESP32 会自动连到手机热点，并访问「电脑 IP:12345」的服务端。

在电脑上若看到服务端窗口里出现来自 ESP32 的请求日志，说明已经连通。

---

### 7.9 使用顺序小结（以后每次用）

1. 手机开热点。
2. 电脑连热点 → 若 IP 有变，重新查 IP 并改 2 个文件里的地址（见 7.4、7.5）。
3. 在 `硬件端/web`（或你放 `app.py` 的目录）运行：`python app.py`，保持不关。
4. 给 ESP32 上电，等待连上热点并访问服务端。

---

### 7.10 可选：减少「电脑 IP 变化」的麻烦

- 部分手机热点支持「**允许客户端使用静态 IP**」或「**绑定设备 MAC 地址**」：在热点设置里给电脑分配固定 IP，这样每次连热点电脑 IP 不变，就不用总改代码。
- 若没有该功能，就每次换网络或重连热点后，按 7.4 重新查 IP 并改一次代码、重新烧录一次。

## 八、接线检查清单（克隆后自检）

1. [ ] WiFi：在 `config.h` 中填好 `WIFI_SSID`、`WIFI_PASSWORD`  
2. [ ] 超声波：Trig→GPIO8，Echo→GPIO18，VCC/GND 正确  
3. [ ] 振动：GPIO3 驱动，蜂鸣器 GPIO11 低电平响  
4. [ ] 按钮：GPIO2 与 GND，按下为低  
5. [ ] 光敏 DO→GPIO10，照明控制→GPIO21  
6. [ ] INMP441：WS=5, SCK=4, SD=6，L/R=GND，供电 3.3V  
7. [ ] MAX98357A：BCLK=15, LRC=16, DIN=7，供电与 GND  
8. [ ] 若接 GPS：改用 Serial1 并修改 `gps.h`，接线 GPS TX → ESP32 RX  
9. [ ] 若用局域网（手机热点+电脑当服务器）：按 **第七章** 改好 **network.cpp、main.cpp** 里的电脑 IP 与端口 12345，并运行 `web/app.py`  
10. [ ] 若自建云服务器：在 `network.cpp`、`main.cpp` 中把 `my_ip` 与端口改为你的服务器地址  
11. [ ] 百度语音：在 `config.h` 中配置自己的百度云语音应用（`BAIDU_CLIENT_ID`、`BAIDU_CLIENT_SECRET`），用于硬件端的**语音识别 + 语音合成**，不经过自建服务端

按上述接线与配置后，用 PlatformIO 编译并烧录到 ESP32-S3，串口监视器 115200 可查看日志与调试信息。

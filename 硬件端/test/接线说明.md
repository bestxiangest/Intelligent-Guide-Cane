# 硬件端接线说明

本文档给出当前主控固件默认使用的核心引脚定义，方便快速接线与排查。

## 主控板

- 开发板：`ESP32-S3 DevKitC-1`
- 工程目录：`硬件端/`

## 主要引脚

| 模块 | 引脚 |
| --- | --- |
| 超声波 Trig | `GPIO8` |
| 超声波 Echo | `GPIO18` |
| 振动模块 | `GPIO3` |
| 蜂鸣器 | `GPIO11` |
| 测试按键 | `GPIO2` |
| 光敏数字输出 | `GPIO10` |
| 照明控制 | `GPIO21` |
| INMP441 WS | `GPIO5` |
| INMP441 SCK | `GPIO4` |
| INMP441 SD | `GPIO6` |
| MAX98357 LRC | `GPIO16` |
| MAX98357 BCLK | `GPIO15` |
| MAX98357 DIN | `GPIO7` |
| GPS RX | `GPIO17` |

## 上电前需要确认

1. 已复制 `硬件端/src/config.local.example.h` 为 `硬件端/src/config.local.h`
2. 已填写 Wi-Fi、百度语音和服务端地址
3. 服务端 `web/` 已能在局域网 `12345` 端口访问
4. 所有模块与 ESP32 共地
5. 如需跳过启动阶段的百度 token 线上获取，可在 `config.local.h` 中填写仍有效的 `BAIDU_ACCESS_TOKEN_SEED` 和 `BAIDU_ACCESS_TOKEN_EXPIRES_AT_SEED`

## 说明

- `SERVER_BASE_URL` 现在统一在 `config.local.h` 中配置，不再直接写死在源码里。
- 百度 `access_token` 会优先从 ESP32 NVS 缓存读取；首次使用 seed token 后也会写入 NVS。
- 本文档不再保留任何真实账号、密码或 API 密钥示例。
